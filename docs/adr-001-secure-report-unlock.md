# ADR-001：服务端报告锁与人工收款解锁

- 状态：已采用
- 日期：2026-06-25

## 背景

完整AI诊断报告属于99元付费内容。旧流程在生成后把完整报告返回浏览器，再用前端页面控制显示，用户可以通过接口响应或浏览器存储绕过付费限制。

## 决策

采用服务端报告保险箱模式：

1. OpenAI生成的完整报告只写入 Turso/SQLite。
2. 客户端只收到报告编号、免费预览和256位随机访问凭证。
3. 报告接口同时校验报告编号和访问凭证。
4. 未付款状态只返回预览，不返回完整报告字段。
5. 管理员确认微信收款后，将报告状态改为 `unlocked`。
6. 客户再次查询时才返回完整报告。

## 取舍

- 优点：无需微信支付商户接口即可真实收款上线；完整报告无法通过普通前端绕过。
- 缺点：收款确认需要管理员人工操作；客户访问凭证保存在当前浏览器，换设备需要后续增加短信、邮箱或访问链接恢复机制。

## 安全措施

- 访问凭证只保存 SHA-256 哈希。
- 管理后台使用签名 JWT、HttpOnly Cookie、issuer/audience 校验。
- 报告、登录和诊断接口设置访问控制与限流。
- 数据库查询使用参数化语句。
- 完整报告响应设置 `Cache-Control: no-store, private`。

## 后续演进

取得微信支付商户资质后，可新增支付订单与回调接口，由支付回调自动执行解锁；现有报告状态模型无需重写。
